## 二、防重复审稿维度（v2.0 新增）

### 背景
2026-06-22 发现全书存在系统性重复文字问题：
- "是最"：ch86-105 高峰期每千字 7-13 次（阈值 ≤3）
- "够了/就够了"：ch86-100 每千字 6-11 次（阈值 ≤3）
- "不是…是"定义式句式：每章 50-80 次
- "不需要"：个别章节 40+ 次
- 递进式句式模板化（"A是最B的→B够了→XX就够了"三段式闭环）

### 新增审稿检查项

**所有审稿 session 必须在报告末尾增加「重复文字密度审计」章节：**

| 检查项 | 阈值 | 工具 |
|--------|------|------|
| `够了`（含"就够了"） | ≤3次/千字CJK | `grep -o "够了" file \| wc -l` |
| `是最` | ≤3次/千字CJK | `grep -o "是最" file \| wc -l` |
| `不需要` | ≤2次/千字CJK | `grep -o "不需要" file \| wc -l` |
| `不是…是`定义式 | ≤5次/千字CJK | `grep -Pc "不是.{0,10}是" file` |
| `就是` | ≤1次/千字CJK | `grep -o "就是" file \| wc -l` |

**判定标准：**
- ✅ 所有指标在阈值内 → 通过
- ⚠️ 1-2 项超标 → 有条件通过（标注超标项，建议返修）
- 🔴 3+ 项超标 → 不通过（必须返修后重新审稿）

### 修复规则（供返修参考）

**"是最"修复方案（轮换使用）：**
- 删除"最"："X是Y的。"
- 换副词："X极其Y" / "X特别Y" / "X无比Y"
- 比较式："没有比X更Y的了"
- 具体化：把抽象"最重要"改成具体比较
- 整句删除：段落结尾重复强调

**"够了/就够了"修复：**
- 段落结尾废话 → 直接删掉
- 冗余收束 → 删掉或换句号
- 保留每章 10-20 个点睛用的"够了"

**"不是…是"修复：**
- 替代："意味着" / "指向" / "其实是" / 直接陈述
- 整句重构：避免"X不是Y。X是Z。"的机械重复

**绝对禁止：**
- ❌ 不能误删系动词"是"（"X极其Y的"应为"X是极其Y的"）
- ❌ 不能删除完整段落
- ❌ 不能改递进式句法风格骨架
- ❌ 不能改知识内容
- ❌ 破折号每章仍需≤15个

## 四、写作规则同步更新

写作阶段就应遵守重复文字阈值，减少后期返修量：

| 规则 | 阈值 |
|------|------|
| "够了"每章 | ≤20次 |
| "是最"每章 | ≤15次 |
| "不是…是"每章 | ≤30次 |
| "不需要"每章 | ≤10次 |
| 破折号每章 | ≤15个 |
| 每章字数 | ≥5000字 |
