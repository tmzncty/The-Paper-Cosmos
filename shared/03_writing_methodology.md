## 1. 文风定位

### 1.1 目标读者画像

**主读者群**：文学类考研备考者（18-28 岁）

| 维度 | 描述 |
|------|------|
| 知识基础 | 已完成本科文学课程，有碎片化记忆但缺乏系统整合 |
| 阅读场景 | 通勤、睡前、课间——碎片化阅读为主 |
| 核心痛点 | 背诵枯燥、知识点孤立、缺乏叙事记忆锚点 |
| 副读者群 | 轻小说爱好者、文学爱好者、对文学史有兴趣的泛知识群体 |

**关键洞察**：主读者群不是来"学新知识"的——他们是来"把已知的碎片串成项链"的。小说的价值不在于首次讲解，而在于**叙事性记忆加固**。一个角色的命运，比一条名词解释记得更牢。

### 1.2 语言风格

**定位：「深夜食堂」式——轻松底色，偶有深度**

不是搞笑番，也不是史诗巨作。是一种温暖的、略带自嘲的、认真讲故事的调性。

```
❌ 过度轻松："哈哈，李白又喝醉了！太搞笑了！"
❌ 过度严肃："唐代诗歌的繁荣，有着深刻的社会历史根源……"
✅ 目标调性："李白在客栈里第三次把酒洒在了诗稿上。杜甫叹了口气，默默帮他把纸移开。
             这大概是天宝年间最寻常的下午——一个天才在糟蹋自己的肝，
             一个未来会在秋风里写下'八月秋高风怒号'的人，在替他操心。"
```

**风格关键词**：
- **轻松但不轻浮**：可以用梗，但梗服务于人物塑造而非搞笑
- **细节有考据**：时代背景、器物、服饰必须准确——考研读者会挑错
- **情感真挚**：轻小说的壳，但人物的情感弧线要经得起推敲
- **自嘲式旁白**：偶尔打破第四面墙，用现代视角回望历史，产生反差萌

**语言分层**：

| 层次 | 用途 | 风格 |
|------|------|------|
| 叙事层 | 主线故事推进 | 现代轻小说白话，节奏明快 |
| 对话层 | 角色交流 | 符合角色身份的个性化语言 |
| 知识层 | 知识点讲解 | 介于口语和学术之间，可标注 [知识锚点] |
| 旁白层 | 作者评论 | 自嘲、吐槽、跨时代类比 |

### 1.3 叙事视角

**主视角：第三人称限制视角（聚焦主角）**  
**辅助视角：第一人称旁白插入（"笔者按"式知识讲解）**

理由：
- 第三人称限制视角是轻小说的主流，读者接受度高
- 可以自然地只展示主角知道的信息，对知识盲区做戏剧化处理
- 辅助的第一人称旁白可以灵活插入知识讲解，而不破坏叙事沉浸

**视角轮换规则**：
- 主线剧情（80% 篇幅）：第三人称限制视角，紧跟主角
- 知识讲解（15% 篇幅）：第一人称旁白，以「编者按」「笔者注」等形式
- 特殊章节（5%）：可切换到其他角色的视角，用于展示不同知识域

### 1.4 章节结构设计

**基础参数**：

| 参数 | 数值 | 理由 |
|------|------|------|
| 每章字数 | 5,000-8,000 字 | 轻小说标准长度，碎片阅读友好 |
| 每卷章数 | 8-10 章 | 单卷 5-8 万字，轻小说单行本标准 |
| 计划总卷数 | 15-20 卷 | 4526 条知识点需要足够的叙事空间 |
| 每章知识密度 | 10-25 条 | 分层处理（详见第 3 节） |

**卷册结构**：

```
第一卷：入学篇（文学理论基础 + 中国文学史先秦段）
第二卷：试炼篇（两汉魏晋文学 + 文献学入门）
第三卷：远征篇（唐宋文学 + 外国文学古典期）
……
```

每卷的叙事弧线对应一个知识域的「集训」，让读者在追剧情的同时完成一个知识模块的复习。

## 2. 知识融入策略

### 2.1 四种融入方法

#### 方法一：对话融入法（占比建议：35%）

**原理**：通过角色之间的讨论、争论、教学场景引入知识点。

**适用场景**：
- 课堂/讲习场景（自然的教学环境）
- 角色之间的学术争论（用冲突驱动兴趣）
- 师徒/同伴之间的知识传授（用关系驱动记忆）

**示例**：

> 「你连这个都不知道？」夜莺翻了个白眼，「'建安风骨'——曹操、曹丕、曹植，加上建安七子，他们写的诗有一种共同的气质：慷慨悲凉、刚健有力。为什么？因为他们活在一个死人比活人多的年代。」
>
> 她把地图摊在桌上，指着北方的大片阴影：「看到了吗？这就是东汉末年的北方。'白骨露于野，千里无鸡鸣'——这不是文学夸张，这是曹操亲眼看过的景象。所以建安文学不是文人无病呻吟，是一群在尸山血海里活下来的人，用笔记录自己的时代。」
>
> 「……你刚才说的那句诗，是曹操《蒿里行》里的？」
>
> 「对。考点之一。还有他的《短歌行》——'对酒当歌，人生几何'——你以为这是在写喝酒？这是求贤令。他要的是人才。考试会问你'建安文学的时代特征'，标准答案就是这四个字：慷慨悲凉。但你得知道为什么悲凉。」

**操作要点**：
- 知识点必须服从角色性格——博学的角色讲理论，实践型的角色讲应用
- 避免「教科书体」——不要让角色突然变成教授
- 用争论和反驳来展示不同观点（比如"唐诗 vs 宋词 哪个更好"的争论）
- 对话中的知识点要有**情感温度**——角色对知识要有态度（热爱、厌倦、困惑）

#### 方法二：情节融入法（占比建议：30%）

**原理**：把文学作品的经典情节、文学史上的重大事件变成异世界的故事桥段。

**核心机制**：在异世界中复现文学史的关键节点，让读者在追剧情时自然复习。

**适用场景**：
- 主线剧情的重大转折
- 角色的成长弧线
- 世界观的历史事件

**示例**：

> 烽火台上，姬宫湦举起了火把。
>
> 阿蝉远远地看着，心跳到了嗓子眼——她知道接下来会发生什么。这个男人会为了博一个女人一笑，点燃烽火戏弄天下诸侯。而那个女人，褒姒，此刻正站在城楼上，面无表情地看着远方。
>
> 「这就是'烽火戏诸侯'，」阿蝉在心里默念，「西周灭亡的标志性事件。《史记》记载——但现代史学界对它的真实性有争议。」
>
> 但她没有时间想这些了。因为火把落下的瞬间，她看见褒姒的嘴角动了一下——不是笑，是一种说不清的、像是在看一场早已知道结局的戏的表情。
>
> 「她知道的，」阿蝉突然明白了，「她一直都知道。」

**操作要点**：
- 情节改编要忠于**核心冲突**，可以在细节上自由发挥
- 让主角成为历史事件的**旁观者或参与者**，而非单纯复述者
- 利用「穿越者视角」制造认知差——主角知道历史结局，但无力改变
- 每个情节融入点都应附带至少一个**考点**（事件意义、文学影响、作者生平）

#### 方法三：描写融入法（占比建议：20%）

**原理**：在环境描写、战斗描写、心理描写中嵌入文学理论和修辞知识。

**适用场景**：
- 战斗/技能系统的设计（以文学理论为基础）
- 环境描写中展示修辞手法
- 角色能力的解说

**示例**：

> 对面的术士举起了法杖。阿蝉感觉到空气中的能量开始以一种奇怪的模式重组——不是直线，不是螺旋，而是一种……跳跃式的连接。
>
> 「通感，」她脱口而出。
>
> 夜莺在旁边低声确认：「没错。他在用钱钟书说的'通感'手法施法——把视觉信号转化为听觉冲击。你看到的颜色，会变成你听到的声音。红色是尖锐的高频，蓝色是低沉的轰鸣。」
>
> 阿蝉还没来得及消化这个信息，世界就变了——她看见一片金黄色的光铺天盖地涌来，然后耳膜像是被一把大提琴的最低音贯穿了。
>
> 「……视觉听觉化。通感。」她咬着牙站起来，「我记住了。」

**操作要点**：
- 把**修辞手法**变成可感知的物理效果（比喻 = 能量形态变化，通感 = 感官交叉）
- 把**文学理论**变成世界观的底层规则（叙事学 = 世界的运行法则）
- 描写中嵌入的知识点应该是**可感知的**，不是纯概念的
- 避免在紧张场景中插入大段讲解——用一句话点到即止，后续再展开

#### 方法四：旁白融入法（占比建议：10%）

**原理**：以作者旁白、编者按、脚注等形式直接讲解知识点。

**适用场景**：
- 复杂概念的系统性讲解（无法自然融入对话）
- 跨时代对比、总结性内容
- 读者可能需要「暂停」消化的重难点

**示例**：

> ※ 编者按：上一节阿蝉提到的"建安风骨"，在考研中属于高频考点。这里补充一个容易被忽略的细节——刘勰在《文心雕龙·时序》中对建安文学的评价是"观其时文，雅好慷慨"，这个"慷慨"不是今天说的"大方"，而是"感慨激昂"的意思。考试如果出名词解释，记得把刘勰这句话引上。

**操作要点**：
- 旁白要**明确标识**（用 ※、编者按、[知识锚点] 等标记），和正文叙事区分
- 旁白的语气可以比正文更直接、更口语化，像是「学习伙伴在旁边提醒你」
- 每章最多 1-2 处旁白，避免打断阅读节奏
- 旁白中可以包含**记忆口诀**、**考试技巧**、**易混辨析**等实用内容
- 旁白位置应放在相关情节**之后**（先建立感性认知，再给理性总结）

### 2.2 方法组合策略

单章内不只用一种方法。推荐的组合方式：

```
开篇（前 1/4）：情节引入 —— 用故事钩住读者
中段（1/4-3/4）：对话 + 描写交替 —— 在叙事中密集植入知识
高潮（3/4 处）：情节融入 —— 让知识点变成剧情转折点
收尾（最后 1/4）：旁白总结 + 预告 —— 给一个「编者按」，埋一个下章悬念
```

### 2.3 知识融入的「三层漏斗」

不是每个知识点都需要同样篇幅。按重要性分三层：

| 层级 | 占比 | 处理方式 | 每章数量 | 示例 |
|------|------|----------|----------|------|
| **核心层** | ~15%（约 680 条） | 专设场景，反复提及，角色命运相关 | 3-5 条 | 李白生平、《红楼梦》结构、现实主义理论 |
| **重点层** | ~35%（约 1584 条） | 融入对话/情节，有 1-2 句解释 | 8-12 条 | 建安七子、《源氏物语》、俄国形式主义 |
| **提及层** | ~50%（约 2262 条） | 一句话带过、背景提及、列表形式 | 10-15 条 | 具体作品的篇名、次要作家、文献版本号 |

**核心层**的知识点会成为角色技能、剧情事件、世界观设定的基石。  
**重点层**的知识点会在对话和描写中自然出现。  
**提及层**的知识点会以「顺便提一句」的方式覆盖——但确保读者能看到这些名词。

## 3. 知识点密度控制

### 3.1 数学模型

```
总知识点：4526 条
计划总章数：约 240 章（15 卷 × 16 章）
平均每章需覆盖：4526 ÷ 240 ≈ 19 条/章
每章字数：5000-8000 字
每条知识点平均占篇幅：300-400 字（含情节包装）
```

**实际分布**（非均匀）：

| 章节类型 | 知识点密度 | 说明 |
|----------|------------|------|
| 高密度知识章 | 30-40 条/章 | 课堂、考试、集中学习场景 |
| 标准叙事章 | 15-25 条/章 | 主线推进中自然融入 |
| 低密度剧情章 | 5-10 条/章 | 高潮战斗、情感转折、纯叙事 |
| 复习/总结章 | 50+ 条/章 | 以复习为主，穿插少量剧情 |

### 3.2 避免「说教感」的六条军规

1. **永远先给故事，后给知识**。读者必须先关心角色，才愿意听角色讲解。
2. **知识讲解不超过连续 300 字**。超过 300 字的理论讲解，必须被对话或情节打断。
3. **每个知识点必须有「情感标签」**。不是"《诗经》分为风雅颂"，而是"夜莺第一次露出真正的笑容，是在读到《关雎》的时候——她说，三千年前就有人知道什么是爱而不得了"。
4. **用冲突驱动知识**。两个角色争论一个问题的两种解释，比一个角色单方面讲解有趣十倍。
5. **允许主角不懂**。主角是考研备考生，不是全知全能的神。让主角犯错、困惑、记混——读者会有共鸣。
6. **知识讲解后必须有回报**。角色运用刚学到的知识解决了问题、通过了考试、赢得了尊重——知识的价值必须被叙事证明。

### 3.3 故事推进与知识讲解的平衡节奏

采用「三幕式 + 知识波峰」结构：

```
一章之内：

┌─ 开头（1000 字）── 叙事钩子，上一章悬念延续
│  ★ 知识密度：低（1-2 条提及层）
│
├─ 上升段（2000 字）── 事件展开，角色互动
│  ★ 知识密度：中（5-8 条重点层 + 提及层）
│
├─ 知识波峰（1500 字）── 集中讲解场景（对话/课堂/争论）
│  ★ 知识密度：高（8-12 条核心层 + 重点层）
│
├─ 下降段（2000 字）── 知识应用，剧情推进
│  ★ 知识密度：中（5-8 条，以应用形式出现）
│
└─ 结尾（1000 字）── 悬念/情感收束，下章预告
   ★ 知识密度：低（1-2 条，埋伏笔用）
```

**关键原则**：知识波峰前后必须有足够的情感铺垫和叙事回报。波峰是读者愿意「暂停剧情听讲解」的前提——因为他们在乎接下来会发生什么。

## 4. 章节模板设计

### 4.1 标准章节模板

```
═══════════════════════════════════════════════════
  第 X 章 《章节标题》
  知识域标注：[中国文学史·唐代] [文学理论·意象说]
  知识点覆盖：核心 3 条 / 重点 8 条 / 提及 6 条 = 17 条
═══════════════════════════════════════════════════

【A. 叙事钩子】（~1000 字）
│
│  · 承接上章悬念
│  · 建立本章核心冲突
│  · 提及层知识点可在此处自然带出
│
│  [适合融入：提及层——通过环境描写、角色回忆带出名词]
│
├──【B. 事件展开】（~2000 字）
│  │
│  │  · 角色互动，推进情节
│  │  · 通过对话自然引入知识
│  │  · 冲突升级
│  │
│  │  [适合融入：重点层——角色讨论、争论、教学]
│  │  [融入方法：对话融入法 60% + 描写融入法 40%]
│  │
│  ├──【B1. 小知识锚点】（可选，~200 字）
│  │    · 对话中某个角色补充的一条背景知识
│  │    · 自然、简短，不打断节奏
│  │
│  └──【B2. 描写中的理论嵌入】（可选，~300 字）
│       · 在环境/战斗描写中展示修辞手法或文学理论
│       · 读者可能需要「二次阅读」才能完全理解
│
├──【C. 知识波峰】（~1500 字）
│  │
│  │  · 本章的知识密度最高点
│  │  · 可以是课堂场景、图书馆查阅、角色间的深度讨论
│  │  · 核心层知识点在此集中讲解
│  │  · 情感上需要一个「小高潮」——角色在知识中发现什么重要线索
│  │
│  │  [适合融入：核心层 + 重点层]
│  │  [融入方法：对话融入法 50% + 情节融入法 30% + 旁白融入法 20%]
│  │
│  └──【C1. 编者按】（可选，~150 字）
│       · 知识波峰结束后的旁白补充
│       · 包含记忆口诀、考试技巧、易错辨析
│       · 用明确标识与正文区分
│
├──【D. 知识应用】（~2000 字）
│  │
│  │  · 角色运用刚学到的知识解决问题
│  │  · 知识从「概念」变成「能力」
│  │  · 推进主线剧情
│  │
│  │  [适合融入：核心层的应用 + 重点层的实践]
│  │  [融入方法：情节融入法 60% + 描写融入法 30% + 对话融入法 10%]
│  │
│  └──【D1. 失败/意外】（可选）
│       · 知识运用出现偏差，引出新的学习需求
│       · 为下一章的知识点埋下伏笔
│
└──【E. 收束与预告】（~500 字）
   │
   │  · 本章情感收束
   │  · 1-2 句提及下一个知识域的伏笔
   │  · 章末悬念
   │
   │  [适合融入：提及层——为下章知识域做预告]
   │
   └──【章末知识清单】（附录，不计入正文）
        · 本章覆盖的知识点列表（供读者自查）
        · 标注：核心 ★ / 重点 ● / 提及 ○
        · 关联真题编号（如有）

═══════════════════════════════════════════════════
```

### 4.2 特殊章节类型

除了标准章节外，还需要以下特殊章节类型：

#### 复习章（每卷末章）

```
结构：剧情中的「阶段考核」场景
· 前半：角色面对考核/考试，回顾本卷学过的内容
· 后半：考核中出现意外，引出下一卷的知识域
· 知识点密度：极高（50+ 条），但以「回忆闪回」形式呈现
· 功能：系统性回顾，帮助读者建立知识框架
```

#### 过渡章（卷间或重大剧情转折前）

```
结构：日常 + 角色发展
· 低知识密度（5-10 条），以情感和人物关系为主
· 功能：让读者「呼吸」，避免知识疲劳
· 可以是温泉回、祭典回、日常训练回
```

#### 外传章（可选，穿插在主线中）

```
结构：从其他角色视角讲述的故事
· 展示不同知识域（如从外国文学角色的视角）
· 功能：扩展世界观，覆盖主线无法触及的知识点
· 频率：每 3-4 章插入一个外传
```

### 4.3 章节标题设计

每章标题应暗含一个知识点线索，既是剧透也是记忆锚点：


## 5. 翻译考量

### 5.1 翻译难点预判

#### 难点一：文学术语的英汉对应

许多中文文学术语在英文中没有完美对应词，需要策略性处理：

| 中文术语 | 英文处理策略 | 示例 |
|----------|-------------|------|
| 意境 | 保留拼音 + 解释："yijing (poetic mood/ambiance)" | 首次出现时用全称，后续可简写 |
| 风骨 | 意译 + 保留拼音："fenggu (literary vigor)" | 结合语境选择 vigor/bone/spirit |
| 建安风骨 | 专有名词，不翻译："Jian'an fenggu" | 括号注明 "the poetic vigor of the Jian'an era" |
| 古文运动 | 直译："Classical Prose Movement" | 已有学术惯例 |
| 文以载道 | 保留原文 + 解释："wen yi zai dao — literature as a vehicle for moral truth" | |

**策略**：
- 首次出现时用「拼音 + 英文释义 + 中文原文」三联标注
- 建立专有术语表（glossary），放在每卷末尾
- 对于无法直译的概念，在英文版中增加脚注

#### 难点二：古诗词的翻译

古诗词是最大的翻译挑战。策略分三层：

| 层级 | 处理方式 | 适用情况 |
|------|----------|----------|
| **学术层** | 直译 + 注释 | 知识讲解场景中的诗句引用 |
| **叙事层** | 意译/改写 | 角色对话中自然引用的诗句 |
| **氛围层** | 英文韵律诗改写 | 环境描写中作为「氛围元素」的诗句 |

**示例**：

> 中文原文：「她低声吟道：'前不见古人，后不见来者。念天地之悠悠，独怆然而涕下。'」
>
> 英文学术层："She murmured: 'Before me, no ancients; behind me, no followers. Contemplating the vastness of heaven and earth, I weep alone in sorrow.'" (Chen Zi'ang, "On Youzhou Terrace")
>
> 英文叙事层："She whispered the old poet's words — something about being alone in all of time, with no one before and no one after. The loneliness of it hit harder than any translation could capture."

#### 难点三：双关、典故、文化负载词

| 类型 | 处理策略 |
|------|----------|
| 谐音双关 | 英文版用脚注解释，或替换为英文中类似的修辞效果 |
| 历史典故 | 简化为英文读者可理解的类比，脚注提供完整信息 |
| 文化负载词 | 如"江湖""风月""红颜"——保留拼音，逐步建立读者认知 |

### 5.2 知识点在翻译中的准确性

**核心原则：知识准确性 > 文学美感 > 语言流畅**

翻译时的优先级：
1. 知识点的**学术准确性**必须 100% 保持——这是考研工具书的底线
2. 文学术语的翻译必须与英文学术界的通行译法一致（参考 *Cambridge History of Chinese Literature* 等权威译本）
3. 当学术准确性和文学美感冲突时，**优先保学术**，用脚注补充文学性翻译

**术语一致性策略**：
- 建立项目级 `glossary.json`，统一所有术语的英文翻译
- 每个术语条目包含：中文、拼音、英文首选译法、备选译法、首次出现位置
- 翻译团队必须查阅 glossary 后才能动笔

### 5.3 中英文版的差异化处理

中英文版不是简单的翻译关系，而是**面向不同读者的两个版本**：

| 维度 | 中文版 | 英文版 |
|------|--------|--------|
| 知识讲解深度 | 可以更深入，读者有基础 | 需要更多背景铺垫 |
| 古诗词引用 | 原文引用，读者可直接理解 | 需要翻译 + 注释 |
| 文化背景 | 假设读者了解基本文化语境 | 需要额外的文化脚注 |
| 旁白注释 | 可以更简洁 | 可能需要更详细的解释 |
| 章末知识清单 | 保持原样 | 需要双语对照 |

**英文版额外增加的内容**：
- 每卷开头的 "Cultural Primer"（文化入门）
- 术语表（Glossary）
- 人名/地名对照表
- 关键概念的深度脚注

## 6. 参考案例分析

### 6.1 《Fate》系列——历史人物角色化

**借鉴要点**：
- 历史/传说人物的「现代转译」：不是复刻历史人物，而是提取其核心特质重新包装
- 能力系统与历史事迹挂钩：Saber 的能力来源于亚瑟王传说，这本身就是一种「知识复习」
- 角色之间的历史关系保留：吉尔伽美什和恩奇都的关系在 Fate 中被忠实保留

**应用到本项目**：
- 将文学史上的关键人物（李白、杜甫、苏轼、莎士比亚、但丁等）设计为角色
- 角色的「能力」来源于其文学成就——比如李白的能力与「浪漫主义」相关
- 保留历史人物之间的真实关系（如李杜的友谊、元白的唱和）

### 6.2 《文豪野犬》——作家能力化

**借鉴要点**：
- 作家的代表作变成超能力名——这是一种极致的知识融入
- 但能力设计需要对原作有深刻理解——否则会变成肤浅的 cos
- 每个角色出场都自带「文学知识彩蛋」，增加读者的发现乐趣

**应用到本项目**：
- 可以设计一个「文学之力」系统：角色的能力名来源于经典作品/文学概念
- 但要避免过度简化——不能让「李白 = 剑客」这样的单一映射
- 每个能力背后要有完整的文学理论支撑，而不是只借用一个名字

### 6.3 《图书馆战争》——概念军事化

**借鉴要点**：
- 把「图书馆管理」这个枯燥概念变成了一场战争
- 每一条图书馆规则都变成了军事条例——枯燥变成了热血
- 但核心始终是「保护知识」的价值观

**应用到本项目**：
- 把「考研复习」这个枯燥过程变成一场异世界冒险
- 每一条知识点都变成了「战斗技能」或「生存知识」
- 核心价值观：知识是有力量的，文学是有温度的

### 6.4 《学园奶爸》——日常中融入知识

**借鉴要点**：
- 日常场景中自然融入育儿知识，不着痕迹
- 知识点不是「被讲解」的，而是角色「正在实践」的
- 温暖的日常感是读者愿意接受知识的前提

**应用到本项目**：
- 不是所有章节都需要宏大叙事——日常中也可以融入知识
- 角色在吃饭、闲逛、争论中自然提到的知识点，往往记得更牢
- 保持「日常感」是长期连载的关键

## 附录 A：快速决策树

写作时遇到「这个知识点怎么融入？」的问题时，按以下决策树操作：

```
这个知识点是核心层吗？
├── 是 → 情节融入法（设独立场景）
│        └── 需要大段讲解吗？
│            ├── 是 → 知识波峰（C 段）+ 编者按
│            └── 否 → 对话融入法（B 段角色讨论）
│
└── 否 → 这个知识点适合对话吗？
    ├── 是 → 对话融入法（B 段）
    │        └── 角色之间有分歧吗？
    │            ├── 是 → 争论场景（增加戏剧性）
    │            └── 否 → 教学/闲聊场景
    │
    └── 否 → 这个知识点可以变成描写吗？
        ├── 是 → 描写融入法（D 段战斗/环境描写）
        └── 否 → 旁白融入法（C1 编者按）
                 或提及层处理（一句话带过）
```

## 附录 B：写作禁区清单

以下做法**绝对禁止**：

| 编号 | 禁区 | 为什么 |
|------|------|--------|
| W1 | 连续超过 500 字的知识讲解无叙事打断 | 读者会跳过 |
| W2 | 角色突然变成教科书说话 | 破坏沉浸感 |
| W3 | 为凑知识点牺牲剧情逻辑 | 读者会弃书 |
| W4 | 知识点出现但没有任何叙事锚点 | 白写了——读者记不住 |
| W5 | 所有角色都像同一个人在说话 | 角色要有个体差异 |
| W6 | 英文版直接音译不解释 | 英文读者会困惑 |
| W7 | 复习章变成纯知识点列表 | 仍然是小说，不是题库 |
| W8 | 忽略知识点之间的关联性 | 孤立的知识点没有复习价值 |
